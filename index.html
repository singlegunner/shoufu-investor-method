一、投资哲学

核心理念

翻倍概率 = 驱动力强度 ÷ 体量约束。寻找翻N倍而非稳健收益的机会。AI是十年结构性机会，产业链全扫描找最大弹性。数据驱动，偏好中小盘。不做预测，做情景推演。

十个信条

① 买在分歧，卖在一致

市场共识形成时，超额收益已消失。真正超额在"大多数人还没看明白"的阶段。

② 仓位=你的观点

仓位比例就是你对自己判断的信心程度。不敢下重注的"看好"没有意义。

③ 先看风险再看收益

任何标的第一反应不是"能涨多少"，而是"下跌空间多大、逻辑破在哪"。

④ 不拍脑袋

所有判断必须有数据支撑。不确定的事明确说"不确定"。

⑤ 体系>直觉

CAN SLIM+PE+PEG框架约束自己。单独任何一个指标都有盲区，体系互补。

⑥ 买多高决定赚多少

估值是收益率的第一决定因素。好公司也要有好价格。

⑦ 不做左侧硬扛

趋势向下时不做"越跌越买"。等右侧信号明确再入场。

⑧ 区分噪音和信号

股价波动是噪音。基本面支柱是否动摇才是信号。

⑨ 情绪操作只能犯一个错

恐慌卖完，别再追高买回来。一个方向错了，反方向操作往往是错两次。

⑩ 时间在你这边

好的基本面+合理的价格=时间站在你这边。便宜买了好公司，短期下跌反而是机会。

个人认知边界

  • 劣势：没有信任背书、没有业绩记录，需要时间积累
  • 不做：不预测短期股价、不给具体买卖建议、不推荐灰色产品
  • 能力圈：A股全市场扫描AI产业链、周期（有色/钴/镍）、光模块、存储、半导体
  • 相信什么：方法论比结果更重要。筛选→检查→情绪控制三闭环跑通，收益是副产品

二、CAN SLIM 初筛体系

来源：William O'Neil，《笑傲股市》

A股适配版参数。不自己做评分模型，用成熟框架。

七字母解读

字母含义A股参数数据源
C当季盈利EPS同比≥20%，营收增速≥25%东财API/腾讯行情
A年度盈利3年ROE≥12%（A股版，原版17%）东财API
N新变化新产品/新管理/新行业趋势→需人工判断影响大小公告+新闻
S供需关系流通市值20-800亿（翻倍弹性区间）腾讯行情API
L领导者RS排名前20%或行业龙头60日涨跌幅计算
I机构认同机构持股增加（可选验证）基金季报
M市场方向大盘不是单边下跌市沪深300均线

A股适配过滤条件

  • 排除ST/*ST
  • 营收增速≥25%（最近季度YOY，用2年CAGR校正）
  • PE>0且≤100x
  • 市值50-800亿（弹性区间）
  • 非金融/非房地产
  • 营收≤200%（排除低基数虚增）

龙头评分（L维度）

  • 龙一：+3分
  • 龙二：+2分
  • 龙三：+1分
  • 非龙头：0分

N（新变化）评分规则

  • N+（大变化）：新爆款、新市场、行业巨变 → 10分
  • N（中等变化）：产能投产、收购、新客户 → 8分
  • N-（日常变化）：常规经营变化 → 5分
  • N/A：无变化 → 3分

M（市场方向）判断

  • 沪深300站稳20日线上翘 → 可操作
  • 在50日线下 → 暂缓

⚠️ 核心修正（相对原版）

不能用单季营收增速代替2年CAGR。 2026-05-06实战发现：300只候选股中PEG<0.3的约20只，但拉2024Q1数据算2年CAGR后，90%的"便宜"来自低基数实际PEG>1.0。

2年CAGR公式：(本期营收/2年前同期营收)^0.5-1，用2026Q1和2024Q1的数据。

阶段3：翻倍类型分流评分

不同类型标的的翻倍驱动力不同，不能用同一套权重：

周期型权重：增长25% / 估值25% / 盈利10% / 利润质量15% / 催化剂15% / 体量10%

阶段4：翻倍路径推演

路径A（双击）

PE修复到行业均值 × 业绩增长50%
公式: 1.5×(avg_pe/pe) ≥ 1.8倍 ✅
适用：低估+业绩改善型

路径B（业绩驱动）

利润翻倍驱动估值不变
净利YOY≥100%=2分，≥50%=1分
适用：高增速成长型

路径C（估值修复）

利润+50%，PE+33%
PE<行业均值×0.7=2分
适用：周期底部+赛道复苏

已知局限性

  • 硬阈值误伤：营收YOY<30%但深耕利基市场的标的被排除（通富微电22.8%、中科曙光23.7%）
  • PE>100的科技龙头：海光(270x)、寒武纪(315x)、天孚(129x)被排除，但赛道逻辑独立成立
  • 体量约束：万亿市值翻倍=涨1万亿，概率极低。同等条件小市值优先
  • 基数效应：单季YOY受去年基数极大影响，必须对比去年同期的增速判断趋势

四、PEG估值方法论

来源：彼得·林奇，《彼得·林奇的成功投资》

PEG是选股框架中的核心估值指标。但PEG不是所有股票都适用。

正确公式

PEG = PE ÷ (增长率数字 + 股息率数字)
  • 增长率：林奇原版用5年预期EPS增长率。没有分析师预测时，用2年EPS CAGR或2年营收CAGR替代
  • 增长率用数字形式：14.8%就用14.8，不是0.148
  • 必须加股息率：林奇原版包含股息率，因为股息也是股东回报的一部分

林奇6种股票分类（使用PEG的大前提）

类型增速区间PEG适用核心关注点
快增长型20-50%✅ PEG<1.0增长可持续性
稳定增长型10-20%✅ PEG<1.5护城河+现金流
缓慢增长型<10%❌ 不适用股息率+回购
周期型波动❌ 不适用商品周期位置
困境反转型—❌ 不适用现金+负债表
资产隐蔽型—❌ 不适用资产重估价值

⚠️ 经典错误：苹果不是稳定增长型

苹果营收+4%，实际是缓慢增长型，不应使用PEG。之前算出PEG=4.66说"偏贵"——结论虽然没错，但方法不对。缓慢增长型应该看股息率+回购.

三种PEG口径

口径公式用途稳定性
PEG(营收)PE÷单季营收YOY筛选器粗筛最稳定⭐⭐⭐
PEG(净利)PE÷单季净利YOY市场常用但周期股误导⭐
PEG(全年)PE÷上全年净利YOY最滞后最平稳⭐⭐

⚠️ PEG口径差异可差10倍+：兆易创新案例

PEG(营收YOY 119%)=108.5/119=0.91 → 合理偏低
PEG(净利YOY 523%)=108.5/523=0.21 → 低基数虚低
PEG(全年49.5%)=108.5/49.5=2.19 → 偏贵
分析时必须同时给出三种口径+标注用的哪种。

PEG评价标准（林奇原版）

PEG评价
<0.5极度低估，千载难逢的便宜货
0.5-1.0优质成长，值得买入
1.0-1.5合理偏高，空间有限
>1.5偏贵，不要买

PEG是买入参考工具，不是卖出信号

卖出决策应该基于基本面变化，而不是"估值到了目标价"。

总回报率法（跨市场对比）

当对比A股/港股/美股不同市场的标的时，PEG不适用（不同市场PE中枢不同）。改用总回报率：

总回报率 = 盈利回报(1/PE) + 回购收益率 + 股息率

跨市场案例：腾讯vs苹果

腾讯：1/17.5=5.7% + 800亿/43500亿=1.8% + 0.95% = 8.5%
苹果：1/33.9=2.9% + 1000亿/41000亿=2.4% + 0.37% = 5.8%

总回报率越高，性价比越高。用这个可以算合理买入价。

五、持有逻辑持续检查 — Thesis Check

核心原则

所有判断基于买入时的具体逻辑支柱，不是泛泛的"公司好不好"。区分噪音（股价波动）和信号（基本面支柱是否动摇）。满仓=没有选择权：保持部分现金是冷静决策的前提。

三步检查流程

Step 1：恢复原始买入逻辑

记录买入时的3-4个具体支柱，必须是可验证的：

  • ❌ "华友钴业是家好公司"
  • ✅ "刚果钴供给收缩→华友自有矿价值翻倍"
  • ✅ "PE 17x在钴周期中是低估区间，过去5年中枢25x"

Step 2：财务数据跟踪

支柱类型检查指标数据来源
业绩增长营收YOY、净利YOYAKShare或快照缓存
盈利改善毛利率趋势同上
估值安全PE变化趋势 vs 买入时腾讯行情API
体量市值变化腾讯行情API

打分：Tracking / Slightly ahead / Slightly behind / Diverging materially

Step 3：定性变化检查

  • 竞争格局：新进入者、客户集中度变化
  • 监管环境：政策变化、出口禁令、关税
  • 管理层：变动、股东行为、内部人交易
  • 业务模型：转型、新业务线、核心业务调整

最终判定

✅ Intact

所有支柱跟踪正常；定性稳定；估值未显著恶化

⬆️ Improved

业绩超预期或逻辑更强了

⬇️ Weakening

一个支柱偏离 + 定性/估值恶化

🚫 Broken

两个以上支柱偏离，或关键定性面不可逆恶化

仓位集中度检查

  • 单只>40% = 极端集中，需充分理由
  • 单只>25% = 高集中度，需定期检查
  • 仓位≠总资产："满仓"概念往往只算主动选股，忘了被动持仓(阿里)和现金。

"二选一"决策框架

核心原则：选哪个都对，关键是选了就别后悔。最差的是两头摆着都不舒服。

  1. 拆解两边的核心矛盾：A便宜但不在风口 vs B贵但在风口
  2. 给三选一：留A（代价踏空）/ 换B（代价波动大）/ 都留（不推荐）
  3. 给出明确验证节点（如"下个财报"），在此之前不动
  4. 不要用"追热点"作为换仓的唯一理由

全市场筛查联动

当需要确认持仓是否调整时，先跑全市场扫描：

  • 持仓 vs 全市场排名：当前持仓在候选池中的相对位置
  • 翻倍路径解读：每只持仓的翻倍驱动力
  • 关键发现：如果没有出现一个明显更好的"一站式替代"标的，不要为了换而换

跨行业标的对比框架

  1. 先比增速基础：营收YOY、净利YOY、毛利率——这是最可比维度
  2. 再看估值：PE高不一定贵，看PEG。PEG<0.5通常便宜
  3. 体量对翻倍难度的影响：同等条件下小市值(<500亿)比大市值(>5000亿)更容易翻倍
  4. 行业周期位置：有色周期底部(估值压缩) vs AI周期高位(估值充分定价)

六、情绪决策系统

⚠️ 已知模式：恐慌→清仓→换仓→后悔

历史案例：3月底卖苹果(255)+减美光+换腾讯(529)。双重伤害——卖在低位买在高位。
2026-05再次出现：卖华友(66)换新易盛(624)。

识别信号

  • 恐慌："中东战争"、"大盘暴跌"、"A股完蛋了"等外部事件驱动
  • 系统性格局判断变形：用"系统性风险"概括所有卖出理由
  • 不甘心："卖飞了XX"、"早知道不卖"、"苹果我已经后悔了"
  • 追热点："光模块涨得猛"、"AI才是未来"等主题驱动
  • "涨不跟跌照跌"综合症：既非板块领涨，跌起来也不落人后
  • 连续操作：一周内2次以上换仓 = 情绪波动期

系统性风险三层判断框架

先区分真假系统性风险，再做预期定价分析，最后检查组合的抗风险能力。

第一层：事件是否改变全球增长路径？

真风险：改变全球经济增长路径（台海、核升级、中美脱钩加剧）→ 需重新评估所有持仓
假风险：局部冲突、短期恐慌（3-6个月修复）→ 恐慌卖出=卖在地板上

第二层："预期定价"分析

高PE本身不危险，危险的是"高预期被打碎"。
真正决定下跌幅度的，是预期被打破的程度，不是PE的数字。

第三层：持仓抗风险能力

是否满仓？是否有现金弹药？是否所有持仓同涨同跌？
满仓=没有选择权 → 焦虑的来源。

第二层详解：预期定价分析（替代简单的PE高低判断）

  • 腾讯PE 16x：市场基本没定价任何增长预期（甚至给了衰退定价）。坏消息来还能跌多少？
  • 新易盛PE 52x：市场定价了年增50%+的持续增长。地缘风险导致AI需求预期下调→双杀
  • 组合分布：低估端（腾讯16x+华友15x）= 安全垫。高估端（新易盛52x+美光36x）= 弹性但脆弱
  • 结论：组合的脆弱性取决于高估端标的的"预期被打碎"概率，不是PE平均值

恐慌时行为准则（贴屏幕）

恐慌时，第一个动作不是卖——是打开这个笔记看今天跌了多少。

1. 如果跌了3%以内 → 不动，去睡觉
2. 如果跌了5%以上 → 看PE：
   PE在历史低位 → 不动
   PE在历史高位 → 减仓1/3，不是清仓
3. 任何时候不卖光 → 卖1/3留2/3
4. 清仓后24小时内不许买任何新标的——让现金躺一夜
5. 减仓和换仓是两件事，分开决策
   要减美光 → 因美光而起
   要补腾讯 → 因腾讯而起
   别因为"腾讯亏了"就去动美光

关键认知

  • 他的痛在"不甘心"（卖飞/追高），不在亏钱本身
  • 已盈利400万+落袋，对亏损容忍度理论上很高，但实际情绪受损严重
  • "仓位" ≠ "总资产"：说满仓时可能是主动持仓满仓，但现金+被动持仓(阿里)仍有大量弹药
  • "换仓"模式识别：切忌把两个标的绑在一起考虑，引导分离为两个独立决策

满仓压力评估

信号判断

🅰 减浮盈最大的（锁利润回笼现金）
🅱 减不在主航道的（换现金等下一次机会）
🅲 不动继续硬扛——明确告知没有余地

三个致命Bug（血泪教训）

Bug 1：批量URL格式

❌ q=sh600519/q=sz000001（多个/q=拼接）→ 返回空数据
✅ q=sh600519,sz000001（逗号分隔一个/q参数）

Bug 2：代码从变量名提取，不从数据体取

批量返回 v_sh603993="1~洛阳钼业~..."
❌ 错误：code = parts[0] → parts[0]='1'（市场代码，不是股票代码）
✅ 正确：re.match(r'v_(?:sh|sz)(\d{6})="', line).group(1) = '603993'

Bug 3：市值单位是"亿"不是"元"

腾讯API：茅台总市值=17193.54（亿），不是1719354000000（元）
❌ df[df['总市值'] > 50e8] → 17193.54 > 50亿 = False，全被过滤
✅ 过滤：df[df['总市值'] > 50]（单位已是亿）

数据校验纪律

每次出数据前，强制三步验证：

  1. 拉API确认财务数据（营收增速/毛利率/净利）
  2. 拉腾讯行情确认行情数据（PE/市值/价格）
  3. 随机抽3-5只熟知的标的做交叉验证

⚠️ 不要凭记忆说数据

随口举例"比亚迪从3500亿涨到8800亿(+560%)"实际上是+151%，被严厉纠正。任何非从数据源实时拉取的数字都不能说。不记得确切数据时要说"这个需要查一下"。

九、组合管理

仓位配置原则

  • 底仓（60-70%）：确定性高的标的
  • 弹性仓（30-40%）：弹性最大的标的
  • 单一标的不超过总仓位40%
  • 同一子行业最多2只，同一大类最多3只

纪律清单

🔴 不做什么（贴在墙上）：
1. 恐慌时不换仓（只减仓，减不超过1/3）
2. 清仓后24小时内不买任何新标的
3. 一周内不做2次以上操作
4. 不减持仓位最轻的去补仓位最重的
5. 不减浮盈最大的去补跌得最惨的

跨市场总回报率法

A股vs港股vs美股，PE中枢不同，PEG不能直接比。改用总回报率：

总回报率 = 盈利回报(1/PE) + 回购收益率 + 股息率

跨市场对比案例：腾讯 vs 苹果

维度腾讯 (PE 17.5x)苹果 (PE 33.9x)
盈利回报1/17.5 = 5.71%1/33.9 = 2.95%
回购回报800亿/43500亿 = 1.84%1000亿/41000亿 = 2.44%
股息回报0.95%0.37%
总回报率8.50%5.76%

回购数据必须从财报原文核实

腾讯2025年年报显示回购约800亿HKD（日均~3.3亿），不是凭印象说的"每天10亿x240天=2400亿"。公司公告数据 > 二手估算。

总回报率越高，性价比越高。用这个可以算合理买入价。

港股特有规则

  • 港元计价：腾讯529是港币不是人民币，不需要做汇率转换
  • 港股通限制：不是所有A股投资者都能买港股通标的
  • 流动性差异：港股权重股流动性好，但中小盘流动性差
  • 财报周期：港股财年和A股一致（1月-12月），但披露节奏不同

美股特有规则

  • 美元计价：实时换算到人民币看总资产
  • 盘前盘后交易：美股有盘前盘后，A股没有，跨市场时差需要考虑
  • SEC报告：美股财报阅读难度高于A股，但AI辅助可弥补
  • 税务：美股股息税30%（非美国居民），港股无股息税

  • 主动持仓 vs 总资产：说"仓位偏高"时可能只算了主动持仓(929万)，忘了被动持仓阿里(150万)+现金(150万)

所有已知陷阱清单

  1. 谷歌API批量查询URL格式：逗号分隔一个/q参数，不是多个/q=拼接
  2. 代码提取：从变量名提取（v_sh603993），不从数据体parts[0]取
  3. 市值单位：腾讯API=亿，不是元
  4. 单季YOY：不能用单季代替2年CAGR，低基数效应虚增
  5. PEG口径：必须标注用的是营收YOY/净利YOY/全年净利，三者可差10倍
  6. AKShare列名：列名会变，先print(list(df.columns))确认
  7. 营收YOY单位：AKShare返回的是"小数百分比"（105.76=105.76%），不是倍数
  8. 验证纪律：输出前随机抽3-5只你熟知的标的交叉验证
  9. 公司公告>二手数据：重大数据去财报原文或官网核实（如腾讯回购800亿不是2400亿）
  10. 不需要自创评分模型：用CAN SLIM+PEG等成熟框架
  11. PEG是买入工具不是卖出信号：卖出看基本面变化，不是估值到目标价
  12. PE高不代表跌得狠：真正决定下跌幅度的是预期被打碎的程度
  13. 不说没有实时验证的数据：不记得的数字说"需要查一下"
